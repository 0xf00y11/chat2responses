## 🎯 核心代码修改与双向同步约束

为确保协议代理逻辑、系统配置、数据模型及 API 端点的一致性，在修改以下模块时，必须严格遵守**链式同步与一致性规则**：

### 1. 🔄 协议转换同步 (Converter ↔ Stream)
*   **约束核心**：改动 `internal/proxy/converter.go`（负责普通 Chat / Responses 的非流式翻译）时，必须**双向同步**修改 `internal/proxy/stream.go`（负责流式及 SSE 协议转换），反之亦然。
*   **研发标准**：确保非流式响应与流式增量（Delta）在处理提示词拼装（如将 System 翻译为 Instructions）、Tool Calls（工具调用参数合并）、Token 统计以及 `reasoning_content`（思考内容）输出时的计算法则与业务表现具有 100% 一致性。

### 2. ⚙️ 配置变更同步 (Config ↔ Schema ↔ Docs)
*   **约束核心**：当对配置管理结构体 `internal/config/config.go` 进行增删改字段时，必须进行**三位一体的链式更新**：
    1.  `config.json.example` —— 必须同步更新字段，并提供合理的默认值与结构。
    2.  `README.md` —— 必须在配置参数表格中对新字段的语义、取值范围、配置建议进行中英文双语或详细中文说明。
*   **研发标准**：严禁出现配置项定义与用户配置文件模板脱节、或者缺失对应文档说明的情况。

### 3. 🗺️ 端点集中注册 (Endpoints Registration)
*   **约束核心**：所有新增的 HTTP 核心代理路由、OAuth2 回调路径或临时服务，必须在 `internal/server/server.go` 的 `RegisterRoutes` 方法中进行**集中式统一管理与注册**。
*   **研发标准**：禁止在各子包内部隐式监听端口，所有路由必须服从 Server 主生命周期的优雅停机（Graceful Shutdown）及错误拦截链。

### 4. 📦 数据模型双向维护 (Data Models)
*   **约束核心**：当修改 `internal/model/` 目录下的底层数据模型时，必须在 `chat.go`（Chat Completions 结构）和 `responses.go`（Responses API 结构）中同步保持序列化（JSON Tag）的统一和对齐。
*   **研发标准**：防止因单侧模型数据类型更改，导致反序列化时发生字段丢失或引起编译器类型报错。

## 🤖 AI 智能体 (Agent) 行为边界与效率约束

在 AI 智能体（Agent）对本项目进行自动分析、代码编写与重构时，须严格遵守以下效率与范围控制指标：

### 1. 📏 单文件物理行数限制 (Single File Size Limit)
*   为了保证代码高可读性、高内聚性、并控制 Agent 会话的上下文（Context）开销，本项目的**单个源文件最大物理行数建议不超过 300 行**。
*   **例外处理机制**：
    *   **最大行数声明**：若在单文件头部添加 `// codex: max-lines(N)` 注释（如 `// codex: max-lines(600)`），则该文件的最大行数限制被重置为 $N$。
    *   **配置文件免检**：所有规则配置文件（如 `AGENTS.md`、`.codex-local.md`、`.codex-rules/*`）天然豁免此行数限制。

### 2. 📁 单次修改文件数限制 (Scope of Change Limit)
*   **单次交互修改的文件总数建议不超过 5 个**。
*   **例外处理机制**：
    *   **规则维护特许**：当对规则体系（如修改 `.codex-rules/` 中的内容、更新项目根目录的 `AGENTS.md`、`.codex-local.md`、以及更新 `init-codex-rules.sh` 脚本）进行维护升级时，不受文件数限制。

### 3. ♻️ 规则维护与版本同步 (Rule Schema Maintenance)
*   **项目场景规则**：修改 `.codex-rules/` 目录中的任何共享场景规则后，必须**同步更新** `init-codex-rules.sh`，保证其他新项目初始化时拿到的规则骨架是最新的。
*   **项目特有规则**：修改 `.codex-local.md` 仅对当前项目生效，无需向规则骨架脚本 `init-codex-rules.sh` 进行同步。
